## Définitions 

- MODELE -> Est un artefact avec ses metadata
- PROJET -> problématique métier
- REPO -> Un modèle, infère la même chose, quand on met un modèle il remplace en théorie le modèle précédent. (au moins en partie
si on fait de l'AB testing). Par exemple un nouveau modèle pour la même raffinerie. 
  - Par exemple: dans un repo, on peut avoir la v1 qui est une reglog, et la v2 qui est un xgboost


# Etapes :
1. Un Projet/repo a été créé
2. Un modèle a été poussé. 
3. L'artefact du modèle a été sauvegardé avec ses metadata
4. La liste des modèles a été consulté.
5. Une nouvelle version du modèle a été poussée
